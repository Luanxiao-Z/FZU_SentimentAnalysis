# 基于当前测试结果的修改方向建议

## 一、现象简要分析

| 类别   | F1      | 观察           |
|--------|--------|----------------|
| 恐惧(4) | **0.934** | 表现最好，可维持 |
| 惊讶(3) | **0.831** | 次优           |
| 开心(1) | 0.782  | 尚可           |
| 厌恶(5) | 0.747  | 尚可           |
| 悲伤(0) | **0.687** | 偏弱，建议重点提升 |
| 生气(2) | **0.595** | 最弱，建议优先改进 |

- **宏平均 F1 ≈ 0.76**：整体有提升空间，主要被「生气」「悲伤」拉低。
- **生气** 的 P/R 都最低，易与 **悲伤、厌恶** 等负面情绪混淆；**悲伤** 次之，与生气、厌恶的边界可能不够清晰。

---

## 二、修改方向（按优先级）

### 1. 诊断：先看混淆矩阵（优先做）

- 在测试集上算并保存 **混淆矩阵**，看「生气」主要被误判成哪几类（大概率是 悲伤/厌恶）。
- 若脚本里还没有，可在 `train_emotion.py` 测试集评估部分加一段：用 `sklearn.metrics.confusion_matrix` 计算，并保存为 `confusion_matrix.json` 或画图保存。
- 根据混淆结果再决定：是加强数据、加约束，还是调损失/权重。

### 2. 数据层面

- **生气 / 悲伤 样本质量与多样性**  
  - 检查 `data_balanced.csv` 里这两类的文本是否过于单一（如句式、用词重复）。  
  - 若有多源数据，可适当增加这两类的样本或做简单增强（同义词替换、回译等），再重新做平衡或按当前比例训练。
- **平衡策略**  
  - 当前已用平衡数据，若之前是用「下采样」做的平衡，可尝试 **轻度上采样** 生气/悲伤（例如 1.2～1.5 倍），而不是严格等量，让模型多见到这两类几轮，再观察 F1。

### 3. 超参与训练策略

- **训练轮数**  
  - 你当前 `--epochs 3` 偏少，建议改为 **4～5**，配合早停（如 patience=2），避免欠拟合，尤其利于难类（生气、悲伤）收敛。  
  - 示例：`--epochs 5`（脚本里若已有 early stopping 会自动停）。
- **学习率**  
  - 当前 `lr=3e-5` 合理；若加长到 4～5 个 epoch 后验证 loss 仍震荡，可试 **2e-5** 更稳。
- **序列长度**  
  - 若很多样本在 128 被截断且后半段有关键情绪词，可试 **max_length=256**（在显存允许下），看生气/悲伤是否略升。
- **weight_decay**  
  - 当前 `0.03` 偏大，若验证集与测试集差距较大（过拟合），可保持；若欠拟合，可试 **0.01**。

### 4. 模型与损失

- **类别权重 / Focal**  
  - 即便用了平衡数据，仍可对 **生气、悲伤** 在损失里略加重（例如在现有类别权重上再乘 1.2～1.5），或使用 Focal Loss 让模型更关注难样本，看 F1 是否提升。
- **分类头正则**  
  - 适当增大分类层 **dropout**（若 BERT 配置里可改），减轻对“简单类”的过拟合，让容量更多用在难类上。

### 5. 输出与复现

- **保存混淆矩阵**  
  - 每次测试集评估后，把 `confusion_matrix` 和 `classification_report` 一并写入 `test_metrics.json` 或单独文件，便于对比不同 run。
- **固定随机种子**  
  - 确保 `train_emotion.py` 里 `RANDOM_STATE` 和 `torch.manual_seed` 等已固定，便于复现和对比“只改某一项”的效果。

---

## 三、建议的下一步执行顺序

1. **加混淆矩阵** → 跑一次当前配置，看生气/悲伤主要和谁混。
2. **epochs 改为 5**（或 4）+ 早停 → 再跑一次，看宏平均与生气/悲伤 F1 是否提升。
3. 若仍不足：在 **数据** 上加强生气/悲伤（多样性/轻度上采样），或 **损失** 上对这两类加重 / 试 Focal，再跑一次对比。
4. 可选：试 **max_length=256**、**lr=2e-5**，做一次消融对比。

---

## 四、可写入脚本的小改动摘要

- 在测试集评估处增加：`confusion_matrix(labels, preds)` 并保存。
- 将 `--epochs` 默认或推荐设为 4～5，并在文档/注释里说明“当前 3 轮偏少”。
- （可选）增加 `--focal_loss` 或 `--class_weights_extra` 等开关，便于做消融实验。

按上述顺序做，一般能先稳住整体、再重点抬升「生气」和「悲伤」的 F1。
