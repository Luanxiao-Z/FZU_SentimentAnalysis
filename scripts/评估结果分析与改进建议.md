# 测试集评估结果分析与模型改进建议

基于运行命令：
```bash
python scripts/train_emotion.py --data data_balanced.csv \
  --output_dir models/ocemotion_best_lr3e5_wd003_ep5 \
  --epochs 5 --batch_size 32 --lr 3e-5 --max_length 128 --weight_decay 0.03
```

## 二、混淆矩阵解读（谁和谁混）

- **生气(2)**  
  - 真实生气 1075 条中：650 判对，**128 误判为悲伤**，**126 误判为厌恶**，91 误判为开心，65 惊讶，15 恐惧。  
  - 即：生气主要与 **悲伤、厌恶** 互混，其次与开心。

- **悲伤(0)**  
  - 真实悲伤 1075 条中：740 判对，**138 误判为生气**，85 开心，63 厌恶，29 惊讶，20 恐惧。  
  - 即：悲伤主要与 **生气** 混，其次与开心、厌恶。

- **厌恶(5)**  
  - 真实厌恶中：88 误判为生气，71 误判为悲伤。  
  - 负面三类「悲伤 / 生气 / 厌恶」之间互相混淆，是当前主要问题。

- **开心(1)**  
  - 与悲伤(75)、生气(65) 有一定混淆，但整体尚可。

- **恐惧(4)、惊讶(3)**  
  - 混淆较少，表现稳定。

**小结**：改进应聚焦「生气 vs 悲伤」「生气 vs 厌恶」「悲伤 vs 厌恶」的边界，而不是全局平均。

## 三、改进建议（按优先级）

### 1. 数据与标注（优先）

- **检查生气/悲伤/厌恶的文本与标注**  
  - 在 `data_balanced.csv` 里抽样查看：  
    - 被标为「生气」但语义更偏「悲伤」或「厌恶」的句子；  
    - 被标为「悲伤」但带明显愤怒表达的句子。  
  - 若有明显标注噪声或边界不清，可做少量清洗或统一标注规范（哪怕只改 5%～10% 的争议样本，有时也能带来 1～2 个点 F1 提升）。

- **数据增强（仅对弱类）**  
  - 对「生气」「悲伤」做轻度增强（同义词替换、回译、或简单复述），使模型多见到这两类的多样表达，而不必动「恐惧」「惊讶」等已足够的类。  
  - 若当前平衡是下采样得到的，可尝试对生气/悲伤 **轻度上采样**（如 1.2～1.3 倍），再训练观察 F1。

### 2. 损失与类别权重（直接针对弱类）

- **对「生气」「悲伤」加重损失权重**  
  - 脚本已支持 `get_class_weights`，可在其基础上对 **label=0（悲伤）和 label=2（生气）** 的权重再乘一个系数（如 1.2～1.5），使梯度更关注这两类。  
  - 实现方式：在 `get_class_weights()` 返回的 `smooth_weights` 上，对对应索引再乘系数即可（可做成 `--boost_weak_classes` 或固定写死 0、2 两类）。

- **尝试 Focal Loss**  
  - 在难样本（如生气、悲伤）上损失更大，有助于拉高其召回。可加 `--focal_loss` 与 `--focal_gamma` 做消融对比。

### 3. 超参与训练策略

- **训练轮数**  
  - 当前 `--epochs 5` 已合理；若验证曲线在 5 轮前已平稳，可保持早停（脚本里已有 `early_stopping_patience=3`）。

- **学习率与正则**  
  - `lr=3e-5`、`weight_decay=0.03` 搭配尚可。若验证集与测试集差距大（过拟合），可保持 0.03；若验证 F1 明显低于测试（或欠拟合），可试 `lr=2e-5`、`weight_decay=0.01` 做一次对比。

- **序列长度**  
  - 若不少样本在 128 被截断且情绪关键词在后半段，可试 `--max_length 256`（在显存允许下），看悲伤/生气是否略升。

### 4. 模型与结构（可选）

- **分类头 Dropout**  
  - 若 BERT 配置中分类头有 dropout，可适当调大（如 0.2～0.3），减轻对「恐惧」「惊讶」等简单类的过拟合，把容量更多留给难类。

- **更強基座**  
  - 在显存与时间允许下，可尝试更大的中文预训练模型（如 `hfl/chinese-roberta-wwm-ext-large` 或当前项目已支持的其它模型），先在小 epoch 上验证是否对生气/悲伤有增益。

---

## 四、建议的下一步执行顺序

1. **确认混淆来源**  
   - 已保存的 `test_metrics.json` 中含 `confusion_matrix`，可据此再细看「生气→悲伤/厌恶」「悲伤→生气」的错例，抽几十条在数据里检查是否标注或表述边界有问题。

2. **先改损失/权重（成本低）**  
   - 在脚本里对悲伤(0)、生气(2) 的 class weight 乘 1.2～1.5，保持其余参数不变，再跑一次；看这两类 F1 和宏平均是否提升。

3. **再动数据**  
   - 若权重调整后提升有限，再考虑：  
     - 对生气/悲伤做数据增强或轻度上采样；  
     - 或清洗/修正部分争议标注。

4. **最后做消融**  
   - 可选：试 `max_length=256`、`lr=2e-5`、`weight_decay=0.01` 或 Focal Loss，各跑一版做对比。

## 五、可写入脚本的小改动摘要

- **类别权重加强**：在 `get_class_weights()` 或调用处，对 `label in (0, 2)` 对应权重乘 1.2～1.5（可加参数 `--boost_sad_anger 1.3`）。
- **Focal Loss（可选）**：增加 `--focal_loss`、`--focal_gamma`，在 `WeightedTrainer.compute_loss` 中分支使用 Focal，便于消融。
- **固定随机种子**：若尚未设置，在 `main()` 开头加 `torch.manual_seed(RANDOM_STATE)`、`np.random.seed(RANDOM_STATE)`，便于复现。
- 混淆矩阵已写入 `test_metrics.json`，无需再改；若要可视化，可另写脚本读该 JSON 画热力图。

按上述顺序做，一般能先稳住整体、再重点抬升「生气」和「悲伤」的 F1，并进一步逼近或超过 0.80 宏平均 F1。
