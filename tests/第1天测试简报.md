## 二、发现的问题

### 严重问题（P0）
1. **[BUG-001] OCR 配置文件路径错误导致多个测试失败**
   - **影响**：`test_extract_text_from_image_path`, `test_extract_text_from_image_bytes`, `test_baidu_api_call_success` 等核心OCR功能测试均因找不到配置文件而失败。根本原因是测试环境缺少 `config/ocr_secrets.toml` 文件，且 `sample_image_path` fixture指向的图片也不存在。
   - **状态**：已定位，待修复

2. **[BUG-002] `test_predict_empty_text_error` 未抛出预期异常**
   - **影响**：模型处理器未能正确处理空文本输入，与预期行为不符。
   - **状态**：已定位，待修复

3. **[BUG-003] `test_baidu_api_call_failure` 报告 `NameError: name 'requests' is not defined`**
   - **影响**：该测试用例本身存在缺陷，在模拟网络请求时未正确导入 `requests` 模块。
   - **状态**：已定位，待修复

4. **[BUG-004] `test_load_settings_from_default_path` 报告 `AttributeError: module 'src.utils.ocr_processor' has no attribute 'tomllib'`**
   - **影响**：在Windows环境下，Python版本兼容性或模块导入方式可能存在问题，导致无法使用 `tomllib` 库。
   - **状态**：已定位，待修复
